The Nearest Centroid Classifier

Each outcome — survived or did not survive — has a centre of gravity in feature space. A new passenger is assigned to whichever centre lies closest. Below is the very algorithm you will implement in Java to train on, and predict from, the real Titanic manifest.

How this maps to your Java code ⟶ 
// Feature vector for one passenger — same encoding as this page.
// sex: female = 1, male = 0.  Pick the features you charted above.
double[] features(Passenger p) {
    return new double[]{ p.getAge(), p.getPclass(), p.getSex() };
}

// ===== TRAIN: a centroid is the mean feature-vector of a class =====
Map<Integer, double[]> train(List<Passenger> data, int dim) {
    Map<Integer, double[]> sum   = new HashMap<>();
    Map<Integer, Integer>  count = new HashMap<>();
    for (Passenger p : data) {
        int label = p.getSurvived();              // 0 or 1
        sum.putIfAbsent(label, new double[dim]);
        count.merge(label, 1, Integer::sum);
        double[] x = features(p);
        for (int i = 0; i < dim; i++) sum.get(label)[i] += x[i];
    }
    Map<Integer, double[]> centroid = new HashMap<>();
    for (Integer label : sum.keySet()) {
        double[] c = sum.get(label); int n = count.get(label);
        for (int i = 0; i < dim; i++) c[i] /= n;   // divide by class size
        centroid.put(label, c);
    }
    return centroid;
}

// ===== PREDICT: label of the nearest centroid (Euclidean) =====
int predict(double[] x, Map<Integer, double[]> centroid) {
    int best = -1; double bestDist = Double.MAX_VALUE;
    for (Integer label : centroid.keySet()) {
        double d = euclidean(x, centroid.get(label));
        if (d < bestDist) { bestDist = d; best = label; }
    }
    return best;
}

double euclidean(double[] a, double[] b) {
    double s = 0;
    for (int i = 0; i < a.length; i++) s += (a[i]-b[i])*(a[i]-b[i]);
    return Math.sqrt(s);
}

⚖ Scaling note for Java: because euclidean() adds every feature's squared difference, a wide-ranging feature (Age) overwhelms a narrow one (Class). Normalise each feature to a comparable range (e.g. (x − min) / (max − min)) before training/predicting — exactly what the “same scale” switch demonstrates here.
